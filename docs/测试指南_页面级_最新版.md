# 智慧综测系统 — 最新测试指南（页面级尽可能详细）

本文档按**页面/操作步骤**组织测试用例，尽量覆盖你们当前仓库的前后端联动功能点。系统由两套前端构成：

- 管理端：`evaluationweb`
- 学生端：`student-submit-web`

后端统一为 `evaluation`。默认后端服务地址按实际环境为准（页面测试主要看 `/api/...` 返回的 `code/message/data`）。

---

## 0. 测试前准备

### 0.1 环境与数据

- 后端服务已启动，MySQL/Redis 可正常访问
- 数据库迁移与种子数据已执行（`evaluation/sql/` 下的脚本按你们当前版本要求执行）
- 管理端与学生端均可通过 Vite 访问后端 API（本地通常是代理到 `/api`）

### 0.2 至少准备的账号（建议）

1. **超级管理员**：权限最全
2. **教师**：已分配负责班级（用于验证数据范围过滤）
3. **学生**：角色为学生，可登录学生端；最好准备至少一个已提交过/一个未提交过的学生

### 0.3 通用观察点（所有页面都适用）

- 错误提示：失败是否给出明确文案（`ElMessage`/弹窗/空状态）
- 加载态：按钮是否有 loading，表格是否有正确 empty 状态
- 空数据：空状态页面是否合理、不报错
- 权限：无权限时侧栏是否不出现；如强行访问 URL 是否被拦截或跳转登录（记录现象）
- Token 过期：管理端/学生端是否在过期后自动清理 token 并重定向到登录页

### 1.2 首页 `/dashboard`（数据分析图“填满屏幕”）

#### 1.2.1 基本渲染

| 序号 | 场景 | 预期 |
|------|------|------|
| D1 | 首次进入首页 | 4 个统计卡片 + 3 个图表区域均可见、无 JS 报错 |
| D2 | 暂无数据 | 图表显示空态（或提示“暂无数据”），不应出现 `undefined`/报错 |
| D3 | 浏览器缩放 | 图表在窗口变化后能够 resize，布局不崩 |

#### 1.2.2 周期筛选

| 序号 | 场景 | 预期 |
|------|------|------|
| D4 | 默认选中周期 | 图表与后端统计口径一致（以你们数据库真实数据为准） |
| D5 | 切换周期 | 统计卡片与图表同步刷新 |
| D6 | 清空周期（若 UI 支持） | 能回到后端“最新启用未归档周期”的口径（以实现为准） |

#### 1.2.3 图表口径校验（重点）

- 申报状态分布图：来自 `evaluation_apply`（同周期）
- 异议状态分布图：来自 `evaluation_objection`（同周期）
- 规则项类型分布：**加分=0，扣分=1**  
  - 图表 x 轴应显示“加分/扣分”而不是 0/1
  - 对应来源为 `evaluation_rule_item.item_type`（按你们现在前端展示逻辑）

### 1.3 系统模块（用户/角色/权限/菜单）

1. `system/user`：分页、筛选、增删改、学院/班级联动（如表单存在）
2. `system/role`：CRUD、为角色分配权限后验证菜单出现/不出现
3. `system/permission`：权限树正确渲染（抽样一致性即可）
4. `system/operation-log`：产生增删改后日志新增、筛选有效

重点：用教师账号验证数据范围过滤是否生效（进入审批/申诉/异议等页面只看负责班级）。

### 1.4 综测 — 规则项管理 `evaluation/item`

#### 1.4.1 分类过滤与展示一致性

| 序号 | 场景 | 预期 |
|------|------|------|
| I1 | 多选分类筛选 | 可多选多个分类并过滤表格内容 |
| I2 | 分类下拉展示完整性 | 不应出现“当前学期暂无规则项”误导；只展示当前周期相关分类 |
| I3 | 跨周期污染 | 不出现非当前周期的分类（例如 `test` 类别不会混入当前过滤） |
| I4 | 分类选项显示文本 | 分类 label 中不出现 `(ID)` 这类混淆信息 |

#### 1.4.2 添加/编辑规则项表单

| 序号 | 场景 | 预期 |
|------|------|------|
| I5 | itemType 为 0/1 | 页面展示与后端一致（加分/扣分语义正确） |
| I6 | moduleCode 自动推断 | 表单不会要求老师手填模块编码；分类变化后能自动推断（以实现为准） |
| I7 | 必填校验 | 必填字段为空时保存失败并提示 |
| I8 | 状态禁用 | 禁用后列表不再展示为启用（学生端可见性应随规则变化而变化） |

#### 1.4.3 复制学期规则（核心重点）

入口：点击工具栏“复制学期规则”对话框

| 序号 | 场景 | 预期 |
|------|------|------|
| C1 | 未选择源/目标周期 | 点击“复制预览/开始复制”无数据提示或禁用（以实现为准） |
| C2 | 查看复制预览 | 预览能显示来源共多少条、目标已有多少条、各分类计数 |
| C3 | 覆盖开关 overwrite 切换 | overwrite=on/off 行为符合：覆盖时目标已有项应被删除/替换（以实现为准） |
| C4 | 开始复制成功 | 复制完成后目标周期规则项列表可见且数量/分类分布符合预期 |
| C5 | 复制后刷新一致性 | 关闭/重新打开页面或切换筛选后数据一致 |

## 3. 跨页面端到端场景（建议整链验收）

说明：同一周期内至少串通一条完整链路，并覆盖异常分支。

### 3.1 正常链路（无异议）

| 步骤 | 操作角色 | 页面/端 | 验证点 |
|------|----------|---------|--------|
| 1 | 管理员/教师 | `evaluation/period-flow` | 配置周期为启用且在各阶段窗口内（未归档） |
| 2 | 管理员/教师 | `evaluation/item` + `evaluation/rule` | 学生端能加载细则项 |
| 3 | 管理员/教师 | `evaluation/academic` | 写入智育分（学业只读） |
| 4 | 管理员/教师 | `evaluation/submit-tip` | 为学生端分区维护提示（后续用于验收） |
| 5 | 学生 | `/app/overview` | 看到流程卡片与当前生效公示 |
| 6 | 学生 | `/app/submit` | 填写并提交申报；动态提示可见 |
| 7 | 教师 | `evaluation/approval` | 看到待审项，执行通过 |
| 8 | 学生 | `/app/records` | 记录状态变为已通过（与管理端一致） |
| 9 | 学生 | `/app/overview` | 在公示窗口内可看到公示；确认无异议成功后入口受限 |

### 3.2 异议链路（公示内提交异议）

| 步骤 | 操作角色 | 页面/端 | 验证点 |
|------|----------|---------|--------|
| 1 | 学生 | `/app/objection` | 异议窗口内可编辑，提交异议成功 |
| 2 | 教师 | `evaluation/objection` | 看到该条待处理；采纳/驳回正确 |
| 3 | 学生 | `/app/records` 或 `/app/overview` | 状态/标签随处理结果变化 |

### 3.3 申诉链路（已驳回项申诉）

| 步骤 | 操作角色 | 页面/端 | 验证点 |
|------|----------|---------|--------|
| 1 | 教师 | `evaluation/approval` | 驳回一条项（产生已驳回状态） |
| 2 | 学生 | `/app/records` | 仅对允许申诉的行出现“申诉”按钮 |
| 3 | 学生 | `/app/records`（弹窗） | 提交申诉理由，状态更新 |
| 4 | 教师 | `evaluation/appeal` | 通过/驳回后学生端状态同步 |

### 3.4 提交无奖项链路（必须验收）

| 步骤 | 操作角色 | 页面/端 | 验证点 |
|------|----------|---------|--------|
| 1 | 学生 | `/app/submit` | 点击“提交无奖项”确认成功；表单清空 |
| 2 | 管理端/教师 | `evaluation/class-score` 未提交名单 | 该学生不出现在未提交名单（以你们口径为准） |
| 3 | 学生 | `/app/records` | 验证页面展示行为是否符合产品预期（注意：若无 apply_item 可能导致记录表空，这属于需验收点） |

### 3.5 Token 过期踢下线

| 编号 | 场景 | 操作与预期 |
|------|------|------------|
| K1 | 管理端 token 到期 | 自动清理 token -> 回到登录页且无法直接访问内页 |
| K2 | 学生端 token 到期 | 自动清理 student_token -> 回到登录页 |

## 4. 权限与菜单对照（便于定位“为什么看不到”）

管理端侧栏出现页面依赖后端权限树 + 前端 `PERM_ROUTE_MAP` 映射。

常见对照（示例）：

- 首页 `/dashboard`：通常为登录后展示（如需权限可按实际）
- 分类/规则/细则项：`sys:category:menu` / `sys:rule:menu` / `sys:item:menu`
- 申报审批：`sys:approval:menu`
- 申诉处理：`sys:appeal:menu`
- 智育：`sys:academic:menu`
- 班级成绩：`sys:class-score:menu`
- 流程与时间：`sys:period:flow:menu`
- 异议处理：`sys:objection:menu`
- 申报提示维护：`sys:submit-tip:menu`

学生端菜单固定为：`/app/overview`、`/app/submit`、`/app/objection`、`/app/records`（由权限接口生成动态路由时确保不出现不允许访问页面）。

## 5. 缺陷记录建议字段（建议测试同学按模板写）

- 端：管理端 / 学生端 + 浏览器版本
- 账号角色：管理员 / 教师 / 学生
- 页面路径：例如 `/dashboard`、`/app/submit`、`/evaluation/objection`

- 操作步骤：按顺序写清楚（1、2、3…）
- 实际结果 vs 预期
- 截图 / 录屏

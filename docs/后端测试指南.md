# 智慧综测系统 — 前端测试指南（页面级）

本文档从**页面与操作**角度说明测试同学应测什么。系统分为两个独立前端，通常配合**同一套后端**（默认 `http://localhost:8080`）使用；本地开发时二者均通过 **Vite 将 `/api` 代理到后端**（管理端默认端口多为 **5173**，学生端在 `vite.config` 中为 **5174**，以实际启动日志为准）。

---

## 0. 测试前准备

### 0.1 环境与数据

| 项目 | 说明 |
|------|------|
| 后端 | 已启动，MySQL / Redis 按 `application.yml` 可用 |
| 数据库 | 已执行项目 `sql/` 下必要迁移（含综测流程时含 `migrate_evaluation_period_workflow.sql`） |
| 管理端 | `evaluationweb` 构建或 `npm run dev` |
| 学生端 | `student-submit-web` 构建或 `npm run dev` |

### 0.2 测试账号建议（至少三套）

1. **超级管理员**：拥有全部菜单权限（含「综测流程与时间」「综测异议处理」等）。  
2. **教师**：拥有审批、申诉、异议、学生管理等按岗位需要的权限，且已在后台配置**所负责的班级**。  
3. **学生**：仅能登录**学生端**；账号在管理端「学生」或用户体系中存在且角色为学生。

### 0.3 通用观察点（所有页面）

- **提示信息**：成功/失败是否有明确 `ElMessage` 或错误文案（勿只盯着无反应）。  
- **加载状态**：表格、按钮是否有 loading，避免重复提交。  
- **空数据**：无数据时表格/空状态是否合理。  
- **权限**：无权限时菜单是否**不出现**；若手动输入 URL 是否跳转登录或空白（与实现一致即可记录为现象）。  
- **登录过期**：Token 失效后是否退回登录页。

## 1. 管理端（evaluationweb）

### 1.1 登录页 `/login`

| 序号 | 操作 | 预期 |
|------|------|------|
| L1 | 账号、密码为空点登录 | 表单校验提示，不发起有效提交 |
| L2 | 错误密码 | 登录失败提示 |
| L3 | 正确管理员账号 | 登录成功进入系统，左侧出现其有权限的菜单树 |
| L4 | 勾选「记住登录」类选项（若有）后刷新 | Token 持久化是否符合产品说明 |
| L5 | 登录成功后浏览器后退到登录页 | 是否再次进入首页或需重新登录（记录现象） |

**说明**：登录表单字段为**用户工号/账号**与密码（对接后端 `studentId` 字段名，教师/管理员同样填账号）。

### 1.2 布局与首页

| 区域 | 测试点 |
|------|--------|
| 顶栏 | 显示当前用户昵称/工号；**退出登录**后回到登录页且无法再访问内页 |
| 侧栏菜单 | 仅展示后端权限树中**有路由映射**的菜单；父级目录仅分组无页面 |
| 首页 `/dashboard` | 文案与快捷入口卡片；点击「用户分配角色」「角色分配菜单」需对应权限才可进入（无权限可无入口或进入后 403） |

### 1.3 系统 — 用户管理 `system/user`（权限 `sys:user:menu`）

| 序号 | 场景 | 建议操作与预期 |
|------|------|----------------|
| U1 | 列表 | 分页、每页条数切换；总数正确 |
| U2 | 查询 | 按工号/姓名等条件筛选（以页面实际筛选项为准） |
| U3 | 新增用户 | 必填校验；成功后列表可见 |
| U4 | 编辑 | 修改后列表/详情一致 |
| U5 | 删除 | 二次确认；删除后列表更新 |
| U6 | 学院/班级联动 | 若表单有学院、班级，切换学院后班级选项是否合理 |
| U7 | 导入教师 Excel | 下载或按说明准备模板；错误行提示；部分成功/全部失败 |
| U8 | 分配角色入口 | 若页面有跳转或弹窗，分配后该用户重新登录菜单是否变化 |

**教师负责班级**（若在用户模块或独立维护入口）：为教师勾选班级后，该教师登录应在「审批 / 申诉 / 异议」等页面**仅看到所带班级学生数据**。

- **为角色分配权限**（若在本页或跳转「角色分配菜单」）：分配后使用该角色的账号登录，侧栏菜单与预期一致。

| 序号 | 场景 | 预期 |
|------|------|------|

| S2 | 学院/班级筛选 | 联动正确 |

### 1.9 系统 — 操作日志 `system/operation-log`（`sys:log:menu`）

- 按时间、操作人、模块等筛选（以页面为准）。  
- 执行几条增删改后，日志是否新增记录（若后端有记日志）。

### 1.10 综测 — 学院 `evaluation/college`（`sys:college:menu`）

### 1.11 综测 — 班级 `evaluation/class`（`sys:class:menu`）

- 班级 CRUD、所属学院、入学年级等字段。

### 1.12 综测 — 细则分类 `evaluation/category`（`sys:category:menu`）

- 分类树或列表与**当前规则**绑定关系正确。  
- 新增/编辑/删除后，「细则项」页签选择分类是否同步。

### 1.14 综测 — 细则项 `evaluation/item`（`sys:item:menu`）

- 在指定规则下维护细则项：名称、模块（德育/学业/素质等）、是否需要材料、`scoreMode`、基础分、系数等。  
- **学业（智育）类**：学生端应对该项不可申报或强提示（与页面「学业由管理端维护」一致）。  
- 列表分页、编辑、删除。

---

### 1.15 综测 — 申报审批 `evaluation/approval`（`sys:approval:menu`）

| 序号 | 场景 | 操作 | 预期 |
|------|------|------|------|
| A1 | 筛选 | 学号、周期、申报状态、**项状态（默认待审）**、学院、班级 | 结果与条件一致 |
| A2 | 教师视角 | 使用教师账号 | **仅本班**学生申报项（与负责班级一致） |
| A3 | 通过 | 选一条待审项，填写备注（若可选） | 项变为已通过；学生端「我的记录」同步 |
| A4 | 驳回 | 填写驳回说明 | 项变为已驳回；学生端可看到驳回并可申诉（在允许时间内） |
| A5 | 审核截止时间 | 周期配置审核截止已过 | 点击通过/驳回应失败并有提示 |
| A6 | 归档周期 | 周期已归档 | 应无法再通过/驳回（有提示） |
| A7 | 分页 | 切换页码、每页条数 | 数据正确 |

### 1.16 综测 — 申诉处理 `evaluation/appeal`（`sys:appeal:menu`）

| 序号 | 场景 | 预期 |

| AP6 | 申诉通过后再次审批 | 在审批页通过/驳回流程完整 |

### 1.19 综测 — 流程与时间（含公示维护）`evaluation/period-flow`（`sys:period:flow:menu`）

## 2. 学生端（student-submit-web）

### 2.1 登录 `/login`

- 使用**学生账号**登录（与管理员账号体系一致，由后台维护）。  
- 错误密码、空字段提示。  
- 登录成功跳转 **`/apply`**。

### 2.2 综测申报工作台 `/apply`（主页面）

页面大致区块：**顶部栏**、**统计卡片**、**流程与公示**、**总分栏**、**按模块 Tab 的申报表单**、**提交栏**、**我的申报记录**、**申诉弹窗**。

#### 2.2.1 顶部与周期

| 序号 | 场景 | 预期 |
|------|------|------|
| ST1 | 周期下拉 | 仅展示**启用中**周期（与后端一致） |
| ST2 | 切换周期 | 细则项、智育分、流程卡片、公示列表随之刷新 |
| ST3 | 刷新记录 / 退出登录 | 退出后需重新登录 |

#### 2.2.2 流程与公示卡片

| 序号 | 场景 | 预期 |
|------|------|------|
| WF1 | 文案 | 申报窗口、公示阶段、异议、审核阶段说明与后台配置时间一致 |
| WF2 | 归档/已确认标签 | 已确认无异议显示标签；归档显示提示 |
| WF3 | 当前生效公示 | 有数据时表格展示起止时间与状态；无数据时空状态 |
| WF4 | 提交异议 | 仅在异议期内可填可点；超期或已确认或归档则禁用 |
| WF5 | 确认无异议 | 二次确认弹窗；成功后按钮与异议入口禁用；无法再申报/申诉/异议 |
| WF6 | 重复确认 | 应有明确错误提示 |

#### 2.2.3 总分栏

- 各模块分值随所选细则项变化；学业显示管理端维护的智育分（只读）。

#### 2.2.4 申报表单（按 Tab：德育、学业、素质等）

| 序号 | 场景 | 预期 |
|------|------|------|
| F1 | 学业 Tab | 明确提示不可申报，仅展示智育只读信息 |
| F2 | 细则项 | 选择细则、备注、材料 URL；「需材料」时必须上传材料行 |
| F3 | 非细则项 | 项目名称、备注必填，材料必填 |
| F4 | 新增/删除行 | 每模块至少保留一行逻辑正确 |
| F5 | 提交申报 | 申报窗口外提交失败有提示；窗口内成功提示并清空草稿区（与实现一致） |
| F6 | 确认无异议后 | 提交申报应失败 |

#### 2.2.5 我的申报记录表格

- 列：申报单、项 ID、状态、申诉摘要、细则项/非细则项、备注、时间。  
- **申诉按钮**：仅已驳回且允许申诉时出现；点击弹窗填理由提交。  
- 审核截止后不可再申诉（有提示）。  
- 申诉审核中/已通过/已驳回状态展示与管理员「申诉处理」一致。

## 4. 权限与菜单对照（便于造数据）

管理端侧栏出现页面依赖后端返回的权限树 + 前端 `PERM_ROUTE_MAP` 映射。常见对应关系：

| 侧栏页面（路径） | 所需权限（示例） |
|------------------|------------------|
| 用户 `system/user` | `sys:user:menu` |
| 角色 `system/role` | `sys:role:menu` |
| 权限 `system/permission` | `sys:perm:menu` |
| 学生 `system/student` | `sys:student:menu` |
| 用户分配角色 `system/user-role` | `sys:user:assign:menu` |
| 角色分配菜单 `system/role-permission` | `sys:role:assign:menu` |
| 操作日志 `system/operation-log` | `sys:log:menu` |
| 学院 / 班级 | `sys:college:menu` / `sys:class:menu` |
